1. What best describes event-driven programming?
Answer: Flow controlled by events like user actions / messages / sensors

2. In a GUI application, which is an event?
Answer: Mouse click

3. What is an event handler?
Answer: A function executed when an event occurs

4. What does the event loop do?
Answer: Waits for events and dispatches handlers

5. Which pattern is commonly used in event-driven systems?
Answer: Observer / Publish-Subscribe

6. A common disadvantage of event-driven systems is:
Answer: Harder debugging due to non-linear flow

7. Which JavaScript API is event-driven?
Answer: addEventListener

8. Message queues are mainly used to:
Answer: Decouple producers and consumers asynchronously

9. Debouncing is used to:
Answer: Reduce how often a handler runs during rapid events

10. Which statement about event handlers is true?
Answer: They may complete later and need careful state handling

11. In Node.js, what manages callback execution order?
Answer: Event Loop

12. A key advantage of event-driven programming is:
Answer: Better responsiveness and loose coupling
